Margin and padding for inline, inline-block and block

Inline

  • Зачастую используется для стилизации текста (жирный, курсив, цвет шрифта и т.д.)
  • Верхние и нижние margin/padding не работают
  • С боковыми отступами можем получить не ожидаемый эффект

Inline-block

  • Поведение характерно для медиа-тегов (img, embed, object)
  • Всегда обтекает содержимое
  • Margin и padding работают ожидаемо

Blosk

  • Используется для разметки
  • Всегда занимает все свободное место по горизонтали у родителя (не всегда это 100% от ширины родителя)
  • У блочных элементов расположенных рядом друг с другом по вертикали наблюдается эффект схлопывания (когда отступы не суммируются, а объединяются между собой). Для отступов слева и справа схлопывание никогда не применяется.
  • Схлопывание не срабатывает:
    • для элементов, у которых на стороне схлопывания задано свойство padding;
    • для элементов, у которых на стороне схлопывания задана граница;
    • на элементах с абсолютным позиционированием, т.е. таких, у которых position установлено как absolute;
    • на плавающих элементах (для них свойство float задано как left или right);
    • для строчных элементов;
    • для '<html>'.

Inline-block верстка - на грани с кроссбраузерностью

Зачем нужен inline-block, когда есть float для блочных элементов? Как было сказано выше, float элементы изымаются из основного потока, что не позволяет нам использовать ряд свойств, например, выровнять "флоатнутый" элемент по вертикале, по отношению к другим элементам, не так и просто.

Inline-block

С другой стороны у inline-block есть недостаток. При обычной разметки страницы между элементами со свойством inline-block появляется пробел и это нормально, так как все же это отчасти строчный элемент. Этот пробел начинает мешать, когда на счету каждый пиксель, мало того, этот пробел в разных браузерах имеет разную ширину.
